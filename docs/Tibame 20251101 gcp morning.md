# GCP 課堂筆記（2025/11/01 上午）

-   逐步帶領同學完成 GCP 免費試用啟用、專案建立與服務開通，建立雲端實作的基本觀念。
-   認識 Cloud Console 的功能分區、搜尋與導航方式，確保後續能快速找到所需資源。

## 帳號與專案設定

-   啟用試用帳號時需綁定信用卡，Google 會先嘗試 1 美元的授權；完成後會取得 300 美元額度（或折算成台幣）。
-   進入 Console 後先確認「專案」與「計費帳戶」關聯無誤，缺少計費綁定會導致服務無法啟用。
-   建議為課程建立獨立專案，未來如需停用可直接刪除專案以避免殘留資源。

## Compute Engine 與網路

-   介紹 Compute Engine 首頁、虛擬機清單與常用欄位（名稱、區域、外部／內部 IP、狀態）。
-   建立 VM 時說明機器型（Series、Machine type）、作業系統映像檔與開機磁碟選項，並比較 Standard 與 Spot VM 的差異。
-   解釋預設 VPC 與內外網 IP 的用途：同專案 VM 之間可透過內網互通，對外連線需依賴外部 IP 與防火牆規則。
-   示範在 Networking／Firewall 中檢視預設規則並新增自訂規則，為後續服務預留所需通訊埠。

## SSH 金鑰與連線方式

-   在本機 `.ssh/` 目錄使用 `ssh-keygen` 產生 key pair，公鑰可貼入 GCP VM 的 `~/.ssh/authorized_keys`。
-   介紹三種上傳公鑰方式：建立 VM 時輸入、進入 VM 後手動寫入、或透過 Project Metadata → SSH Keys 讓所有新 VM 共用同一把鑰匙。
-   以 Browser-based SSH 作為快速登入示例，再演練本機終端機輸入 `ssh -i <private_key> <user>@<external_ip>` 的做法。
-   解釋首次連線出現的 fingerprint 確認訊息會被記錄於 `known_hosts`，更換主機或金鑰時需要重新確認。

## 操作建議與常見問題

-   建議為每台開發用電腦建立獨立的 key pair，避免共用私鑰造成資安風險。
-   沒有預先開防火牆規則時，瀏覽器無法造訪 VM 上的服務；可依需求針對特定埠號與來源 IP 開放。
-   遇到 Console 欄位找不到時，可利用全域搜尋（上方搜尋列）快速定位服務或設定頁面。
-   若忘記 VM 登入帳號，可在 Console 的 VM 清單畫面查看欄位或於機器詳細頁查詢。
